# 0006 — pgx/v5 как драйвер PostgreSQL

### Контекст

Go предоставляет стандартный интерфейс `database/sql` с несколькими драйверами для PostgreSQL. Альтернативы — `lib/pq` (устаревший) и `jackc/pgx`.

## Решение

Используется `jackc/pgx/v5` напрямую, без обёртки `database/sql`.

Причины:

- `pgxpool` предоставляет встроенный пул соединений с настройкой `MaxConns`, `MinConns`, `MaxConnLifetime`, `MaxConnIdleTime` — без сторонних пакетов,
- нативная поддержка PostgreSQL-специфичных типов: `uuid`, `timestamptz`, enum,
- `pgx.ErrNoRows` как явная ошибка отсутствия строки без необходимости проверять `sql.ErrNoRows` через `errors.Is`,
- `pgconn.PgError` позволяет проверять PostgreSQL error code напрямую (`23505` для нарушения уникальности) без разбора строки.

Транзакционность реализована через `TxManager[T]` в `pkg/db/tx.go`: tx-объект передаётся через контекст с помощью `lib-ctxval`, что позволяет адаптерам прозрачно участвовать в транзакции без изменения сигнатур.

### Последствия

- Код не совместим с `database/sql` — смена драйвера потребует переписывания слоя хранилища.
- Прямой доступ к `PgError.Code` упрощает обработку constraint-ошибок.
- `pgxpool` управляет пулом соединений без дополнительных зависимостей.

## Ограничения

Отказ от `database/sql` означает отказ от части экосистемы (например, `sqlx`). Это осознанный компромисс в пользу полной поддержки PostgreSQL-специфики.
