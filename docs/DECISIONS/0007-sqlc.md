# 0007 — sqlc для генерации кода из SQL

### Контекст

Для работы с PostgreSQL существует несколько подходов: ORM (gorm, ent), query builder (squirrel), ручное написание SQL с `pgx`, кодогенерация из SQL.

ORM скрывает SQL и затрудняет оптимизацию запросов. Query builder добавляет DSL поверх SQL. Ручное написание с `pgx` требует вручную описывать структуры для Scan и поддерживать их синхронизацию со схемой.

## Решение

Используется [sqlc](https://sqlc.dev) — генератор Go-кода из SQL-запросов и схемы БД.

SQL-запросы пишутся вручную в `internal/db/queries/*.sql` и аннотируются:

```sql
-- name: GetProductByID :one
SELECT * FROM products WHERE id = $1 AND deleted_at IS NULL;
```

sqlc генерирует типизированные Go-функции, структуры параметров и результатов в `internal/db/gen/`. Сгенерированный код не редактируется вручную и исключён из линтера.

### Последствия

- SQL остаётся единственным источником правды для запросов.
- Компилятор проверяет соответствие типов между Go-кодом и SQL.
- Изменение схемы требует регенерации — несоответствие обнаруживается на этапе `sqlc generate`, а не в рантайме.
- Запросы легко читаются и оптимизируются без знания Go-DSL.

## Ограничения

sqlc не поддерживает динамические запросы с переменным набором условий. Для таких случаев потребуется ручной SQL или query builder.
