# 0010 — chi/v5 как HTTP-роутер

### Контекст

Стандартная библиотека Go (`net/http`) предоставляет базовый `ServeMux` без поддержки параметров пути, групп маршрутов и middleware-цепочек. Для REST API с версионированием и общими middleware это требует ручного кода.

Альтернативы: gin, echo, fiber, gorilla/mux, chi.

## Решение

Используется [go-chi/chi/v5](https://github.com/go-chi/chi).

Причины выбора chi перед альтернативами:

- совместим с `net/http` — хендлеры имеют стандартную сигнатуру `http.HandlerFunc`, нет vendor lock-in на типы фреймворка,
- поддержка групп маршрутов и вложенных роутеров для версионирования (`/api/v1`),
- middleware как стандартные `func(http.Handler) http.Handler` — легко тестировать и переиспользовать,
- минимальная зависимость: только роутер без ORM, валидаторов и прочего.

### Последствия

- Хендлеры совместимы со стандартной библиотекой — их можно тестировать через `httptest` без chi.
- Параметры пути (`chi.URLParam`) доступны только при прохождении запроса через chi-роутер — тесты с параметрами пути требуют `ServeHTTP` через роутер.

## Ограничения

chi не предоставляет встроенной валидации, сериализации или обработки ошибок — это намеренно, эти задачи решаются в других слоях.
