# 0013 — koanf для конфигурации

### Контекст

Конфигурация сервиса загружается из двух источников: хардкодные значения по умолчанию и переменные окружения. Стандартный `os.Getenv` требует ручного парсинга типов и не поддерживает иерархические ключи. `viper` — наиболее распространённая альтернатива — страдает от слабой типизации и объёмного API.

## Решение

Используется [koanf](https://github.com/knadh/koanf) через обёртку `lib-config`.

`lib-config` реализует строгий загрузчик конфигурации с пятью фазами валидации:

1. **struct contract** — каждое экспортируемое поле обязано иметь тег `koanf`,
2. **unknown keys** — нераспознанные ключи репортируются все сразу через `errors.Join`,
3. **decode** — несоответствия типов,
4. **struct tags** — field-level ограничения через `go-playground/validator`,
5. **semantic** — кросс-полевые ограничения через метод `Validate()` на `*T`.

Переменные окружения используют двойное подчёркивание `__` как разделитель вложенности:

```
INIT__DB__DSN → db.dsn
```

`WeaklyTypedInput: false` — `"1"` вместо `int` является ошибкой конфигурации, а не допустимым значением.

### Последствия

- Все ошибки конфигурации выдаются одновременно до старта сервера.
- Опечатка в имени переменной окружения обнаруживается немедленно (unknown key).
- Конфигурация type-safe: нет `viper.GetString` с приведением типов в рантайме.

## Ограничения

Конфигурация поддерживает только два источника: defaults + env vars. Файлы конфигурации не поддерживаются намеренно — в контейнеризированных окружениях env vars являются стандартным механизмом.
